核心概念详解

💾 字符串类型 (string)

在Solidity中,字符串是动态大小的UTF-8编码文本:

string name;  // 状态变量,存储在区块链上
string bio;   // 永久保存,消耗gas

重要特点:

  • 字符串是引用类型,不是值类型
  • 存储在状态变量中是永久的
  • 长字符串存储成本高(每字节消耗gas)
  • 不能直接用==比较,需要使用keccak256哈希
🔑 memory 关键字

memory是Solidity中的数据位置之一,表示临时存储:

function add(string memory _name, string memory _bio) public {
    name = _name;  // 从临时内存复制到永久存储
    bio = _bio;
}

数据位置详解:

  • storage: 永久存储在区块链上(状态变量默认)
  • memory: 临时存储,函数执行完就销毁
  • calldata: 只读的临时存储(更节省gas)
  • 引用类型(string、array、struct)必须显式指定数据位置
👀 view 函数

view修饰符表示函数只读取状态,不修改:

function retrieve() public view returns (string memory, string memory) {
    return (name, bio);  // 只读取,不修改
}

关键优势:

  • 调用view函数不需要发送交易
  • 不消耗gas(从外部调用时)
  • 可以返回数据给调用者
  • 比普通函数更安全(编译器会检查是否修改状态)
💡 提示: 还有pure函数,它既不读也不写状态,更严格!
🎁 多返回值

Solidity允许函数同时返回多个值,这在其他语言中不常见:

function retrieve() public view returns (string memory, string memory) {
    return (name, bio);  // 用括号包裹多个返回值
}

使用方式:

// 调用方接收多个返回值
(string memory userName, string memory userBio) = myContract.retrieve();
  • 返回值类型需要用括号包裹
  • return语句中的值也要用括号包裹
  • 接收时可以使用解构赋值
  • 可以跳过不需要的返回值:(string memory userName, ) = ...

完整代码详解

📄 完整合约代码

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SaveMyName {
    // 状态变量 - 永久存储在区块链上
    string name;
    string bio;
    
    // 添加/更新数据
    function add(string memory _name, string memory _bio) public {
        name = _name;
        bio = _bio;
    }
    
    // 只读函数 - 检索数据
    function retrieve() public view returns (string memory, string memory) {
        return (name, bio);
    }
    
    // 组合函数 - 保存并立即返回
    function saveAndRetrieve(string memory _name, string memory _bio) 
        public returns (string memory, string memory) {
        name = _name;
        bio = _bio;
        return (name, bio);
    }
}
✅ 这个合约做什么?

这是一个简单的个人信息存储合约。你可以保存姓名和简介到区块链上,任何人都可以读取,但只有你能修改(在实际应用中需要加入访问控制)。

🔍 函数详细对比

函数1: add() - 存储数据

function add(string memory _name, string memory _bio) public {
    name = _name;
    bio = _bio;
}
  • 作用: 将新数据写入区块链
  • 消耗: 需要发送交易,消耗gas
  • 参数: 使用memory,表示临时存储
  • 无返回值: 只执行存储操作

函数2: retrieve() - 读取数据

function retrieve() public view returns (string memory, string memory) {
    return (name, bio);
}
  • 作用: 从区块链读取已存储的数据
  • 消耗: 免费!不需要gas(外部调用时)
  • 修饰符: view保证不修改状态
  • 返回值: 同时返回姓名和简介

函数3: saveAndRetrieve() - 存储并返回

function saveAndRetrieve(string memory _name, string memory _bio) 
    public returns (string memory, string memory) {
    name = _name;
    bio = _bio;
    return (name, bio);
}
  • 作用: 存储数据并立即返回确认
  • 消耗: 需要gas(因为修改了状态)
  • 注意: 没有view,因为修改了状态
  • 用途: 一次操作完成存储和确认
⚠️ 重要区别

view函数调用时不消耗gas,但returns只是返回数据的普通函数需要消耗gas!区别在于是否修改了状态。

📚 深入理解

为什么需要memory?

值类型(uint、bool、address等)大小固定,直接复制效率高。
引用类型(string、array、struct)大小可变,直接复制成本高,所以需要指定数据位置。
memory告诉编译器:"这个数据只是临时的,用完就扔掉"。

view vs pure vs normal

函数类型 读状态 写状态 Gas
normal ✅ ✅ 需要
view ✅ ❌ 免费(外部调用)
pure ❌ ❌ 免费(外部调用)
