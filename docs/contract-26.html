核心概念

🎯 NFT市场工作流程
  • 卖家挂单
    授权市场合约操作NFT,设置价格和版税比例
  • 买家购买
    发送ETH,市场合约转移NFT并分配资金
  • 费用分配
    平台抽取手续费,创作者获得版税,卖家获得剩余金额
  • 取消挂单
    卖家可随时取消未售出的挂单
💸 费用分配机制

示例: NFT售价 100 ETH

总价 = 100 ETH
├─ 平台费 (2%) = 2 ETH → 平台
├─ 版税 (10%) = 10 ETH → 卖家(创作者)
└─ 卖家收入 = 88 ETH → 卖家
✅ 公式:
卖家收入 = 总价 - (总价 × 平台费率) - (总价 × 版税费率)
🔑 ListingID 生成机制

使用keccak256哈希确保每个挂单的唯一性:

bytes32 listingId = keccak256(abi.encodePacked(
    nftContract,  // NFT合约地址
    tokenId,      // NFT的ID
    seller        // 卖家地址
));

这样即使同一个NFT被多次挂单,每次的listingId也是唯一的。

🛡️ 安全措施
安全特性 实现方式 防范风险
ReentrancyGuard nonReentrant修饰符 防止重入攻击
购买前删除挂单 delete listings[listingId] 防止双重消费
所有权检查 nft.ownerOf(tokenId) 验证卖家拥有NFT
批准检查 getApproved/isApprovedForAll 确保市场可转移NFT
价格验证 require(msg.value == price) 确保付款金额正确

}

2️⃣ 挂单NFT

function listNFT(
    address nftContract,
    uint256 tokenId,
    uint256 price,
    uint256 royaltyPercent
) external {
    require(price > 0, "Price must be > 0");
    require(royaltyPercent <= 100, "Royalty too high");
    
    // 验证所有权
    IERC721 nft = IERC721(nftContract);
    require(nft.ownerOf(tokenId) == msg.sender, "Not owner");
    
    // 验证批准
    require(
        nft.getApproved(tokenId) == address(this) || 
        nft.isApprovedForAll(msg.sender, address(this)),
        "Not approved"
    );
    
    // 生成唯一ID
    bytes32 listingId = keccak256(abi.encodePacked(nftContract, tokenId, msg.sender));
    
    // 保存挂单
    listings[listingId] = Listing({
        seller: msg.sender,
        nftContract: nftContract,
        tokenId: tokenId,
        price: price,
        royaltyPercent: royaltyPercent
    });
    
    emit Listed(listingId, msg.sender, nftContract, tokenId, price, royaltyPercent);
}
⚠️ 重要: 卖家必须先调用NFT合约的 approve() 或 setApprovalForAll() 授权市场合约!

3️⃣ 购买NFT

function buyNFT(bytes32 listingId) external payable nonReentrant {
    Listing memory listing = listings[listingId];
    require(listing.price > 0, "Not listed");
    require(msg.value == listing.price, "Incorrect price");
    
    // 先删除挂单(防止重入)
    delete listings[listingId];
    
    // 计算费用分配
    uint256 marketplaceFee = (listing.price * marketplaceFeePercent) / 100;
    uint256 royaltyFee = (listing.price * listing.royaltyPercent) / 100;
    uint256 sellerProceeds = listing.price - marketplaceFee - royaltyFee;
    
    // 转移NFT
    IERC721(listing.nftContract).transferFrom(
        listing.seller,
        msg.sender,
        listing.tokenId
    );
    
    // 分配付款
    if (marketplaceFee > 0) {
        payable(feeRecipient).transfer(marketplaceFee);
    }
    
    if (royaltyFee > 0) {
        payable(listing.seller).transfer(royaltyFee);
    }
    
    payable(listing.seller).transfer(sellerProceeds);
    
    emit Purchase(listingId, msg.sender, listing.seller, listing.price);
}
💡 为什么先删除挂单?

在CEI模式(Checks-Effects-Interactions)中:

  1. Checks: require验证
  2. Effects: 修改状态(delete listings)
  3. Interactions: 外部调用(transfer)

先修改状态可防止重入攻击!

4️⃣ 取消挂单

function cancelListing(bytes32 listingId) external {
    Listing memory listing = listings[listingId];
    require(listing.seller == msg.sender, "Not seller");
    
    delete listings[listingId];
    emit Unlisted(listingId);
}

5️⃣ 管理函数

function setMarketplaceFeePercent(uint256 _newFeePercent) external onlyOwner {
    require(_newFeePercent <= 100, "Fee too high");
    marketplaceFeePercent = _newFeePercent;
    emit FeeUpdated(_newFeePercent);
}

function setFeeRecipient(address _newRecipient) external onlyOwner {
    require(_newRecipient != address(0), "Invalid address");
    feeRecipient = _newRecipient;
}

receive() external payable {
